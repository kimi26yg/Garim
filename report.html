Project Overview

Garim Attack (Red Team)

A specialized "Attacker Console" simulating deepfake threats in real-time. Designed to generate sophisticated attacks to test the defense system.

  • Real-time Deepfake Injection
  • WebRTC Peer-to-Peer Video
  • Socket.IO Signaling Control
  • "Cyber/Hacker" Aesthetic UI

Garim Eye (Blue Team)

An on-device defense system leveraging a hybrid AI engine to detect manipulated media with high accuracy and low latency.

  • Hybrid Detection (FFT + CNN)
  • On-device Privacy First
  • Real-time Reliability Scoring
  • Secure Audit Logging

Red Team: Garim Attack

Architecture & Tech Stack

Built with Flutter for cross-platform capability, utilizing a robust stack for high-performance media manipulation.

Flutter Python Server WebRTC Riverpod

The app coordinates three main data streams: Signaling (Connection), Video Streaming (P2P), and AI Processing (Deepfake Gen).

Key Capabilities

  • Deepfake Overlay: Server-side processing renders swapped faces into the live stream.
  • Secure Dialer: Custom keypad interface for initiating "attacks" (calls).
  • Latency Monitoring: Real-time graphs to ensure attack stability.
  • Station Identity: dynamic "Virtual Number" generation.

Blue Team: Garim Eye v2

Reliability Scoring

We don't just output a number. The system uses an Exponential Moving Average (EMA) filter to smooth out noise from individual frames.


Status Thresholds:
  • SAFE (0.0 - 0.4): Normal Activity
  • WARNING (0.4 - 0.7): Potential Anomaly
  • DANGER (0.7 - 1.0): High Probability of Deepfake

Audit Logging

Every detection event is logged securely for post-analysis without compromising user video data.

  • Timestamped Event Logs
  • Raw Score vs Confidence Score
  • Network Latency Records
  • Encrypted Local Storage
